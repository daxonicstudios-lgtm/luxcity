# LuxCity RP — v1.1 Update Roadmap

**Studio:** Daxonic Studios
**Current Version:** 1.0
**Target Version:** 1.1
**Engine:** Three.js r160 (WebGL) — Single-file architecture (`index.html`)
**City:** 100% procedurally generated in code (no external city assets)

---

## Overview

v1.0 is a working open-world RP game with a procedurally generated city, driveable Porsche, animated character, and mobile touch controls. v1.1 focuses on **visual quality**, **city realism**, and **performance** using industry-proven Three.js techniques.

|-----------|--------|--------|

## Phase 1: Trees + Vegetation (Priority: CRITICAL)

### 1.1 Replace Cotton Ball Trees

**Current:** `SphereGeometry(2, 10, 8)` on a `CylinderGeometry` — green ball on a stick.

**New approach — Multi-layered cone trees:**
- 3 stacked `ConeGeometry` layers (large base → small top) with slight random offsets
- Each cone slightly different green hue for depth
- Trunk uses tapered cylinder with slight bend (random lean ±5°)
- **Randomization**: height ±25%, trunk width ±15%, color hue shift ±15%, lean angle

**Tree species:**
- **Oak** — 2-3 overlapping spheres at different offsets creating an irregular canopy
- **Pine** — 3 stacked cones, darker green `0x1a5a1a`, narrow silhouette
- **Birch** — thin white trunk `0xe8dcc8`, small scattered leaf clusters (3-4 small spheres)
- **Palm (improved)** — curved fronds using bent PlaneGeometry with double-sided material

**Bushes:** Replace single sphere with 2-3 overlapping small spheres at random offsets for organic shape.

### 1.2 Grass & Ground Variation

**Current:** Single flat green `PlaneGeometry(2000, 2000)`, color `0x4a6a3a`.

**New approach — Canvas-generated ground texture:**
- Use Canvas2D to paint varied grass: darker/lighter patches, dirt near roads, worn areas near buildings
- Apply as `CanvasTexture` with `wrapS/wrapT = RepeatWrapping`
- Add scattered ground-level planes for flower patches (small colored circles)

## Phase 2: Roads & Infrastructure (Priority: CRITICAL)

### 2.1 Road Detail

**Current:** Flat dark planes (`0x1c1c20`) with only yellow center dashes.

**New approach — Canvas-generated road texture:**
- Generate road texture on Canvas2D:
  - Asphalt base with subtle noise/grain variation
  - White lane dividers (dashed)
  - Yellow center line
  - Crosswalk zebra stripes at intersection edges
  - Turn arrows, stop lines
- Apply as `CanvasTexture` to road mesh — one texture per road segment

**Geometry additions:**
- **Raised curbs**: `BoxGeometry(length, 0.15, 0.3)` along road edges, color `0xc0c0b0`
- **Crosswalks**: White stripe planes at every intersection (5 stripes, 0.5m wide, 3m long)
- **Manhole covers**: `CircleGeometry(0.4)` dark metal, randomly placed on roads

### 2.2 Sidewalk Upgrade

**Current:** Flat gray planes per block.

**New approach:**
- Canvas texture with tile/paving pattern (grid lines every 1m)
- Slightly raised (0.12m) above road level with curb edge
- Different color zones: lighter near buildings, darker near roads

### 2.3 Intersection Detail

- Stop lines before each intersection (solid white plane)
- Pedestrian crosswalks on all 4 sides
- Turn arrows on inner lanes (flat white geometry or canvas-painted)
- Road signs at corners (stop signs, street names)

## Phase 3: Buildings — Canvas Textures (Priority: HIGH)

### 3.1 Canvas-Generated Building Facades

**Current:** Flat `MeshStandardMaterial({ color })` with emissive window strips.

**New approach — per-building Canvas textures:**
```
For each building face:
1. Create Canvas2D at resolution proportional to face size
2. Fill base color (concrete/brick/glass)
3. Draw window grid:
   - Dark rectangles for windows
   - Random brightness per window (some lit, some dark)
   - Window frames (thin lines between windows)
   - Ground floor different (storefronts, doors, awnings)
4. Add surface detail:
   - Horizontal floor separator lines
   - Vertical pilaster lines on some buildings
   - AC units on random windows (small dark rectangles)
5. Apply as CanvasTexture to face material
```

**Building-type variations:**
- **Skyscrapers**: Blue-tinted glass grid, reflective material, setback geometry on top 1/3
- **Apartments**: Warm-toned windows, balcony shadows painted on texture, laundry details
- **Commercial**: Bright ground-floor storefronts, neon-colored sign bands, awning geometry
- **Houses**: Smaller windows, shutters painted on, brick/siding texture patterns

### 3.2 Building Geometry Improvements

- **Skyscraper setbacks**: Top section 70% width of base (add second box on top)
- **Entrance canopies**: Glass `BoxGeometry` extending from ground floor
- **Rooftop variety**: Water towers (cylinder + cone), antenna arrays, AC clusters, helipad circles
- **Ledges/cornices**: Thin `BoxGeometry` strips every 3-4 floors

## Phase 4: Street Props & Furniture (Priority: HIGH)

### 4.1 New Props to Add

| Prop | Geometry | Placement |
|------|----------|-----------|
| **Bollards** | CylinderGeometry(0.1, 0.1, 0.8) | Along sidewalk edges, every 5m |
| **Bus stops** | BoxGeometry shelter + PlaneGeometry sign | 1 per block edge |
| **Mailboxes** | BoxGeometry(0.4, 1, 0.35) blue | Near intersections |
| **Planters** | BoxGeometry + SphereGeometry bush | Outside commercial buildings |
| **Dumpsters** | BoxGeometry(2, 1.5, 1.2) green | Behind buildings |
| **Newspaper stands** | BoxGeometry with angled top | Sidewalks in downtown |
| **Billboards** | Large elevated PlaneGeometry | Along major roads |
| **Road signs** | Octagon (stop) / Rectangle on post | Every intersection |
| **Parking meters** | Thin cylinder + small box top | Along parking areas |

### 4.2 Use InstancedMesh for All Props

- Group all identical props into InstancedMesh batches
- One draw call per prop type (trees, lamps, hydrants, bollards, etc.)
- Expected: 300+ individual meshes → ~15 instanced draw calls

## Phase 5: Lighting & Atmosphere (Priority: HIGH)

### 5.1 Day/Night Cycle

- Sun directional light orbits over 5-10 minute real-time cycle
- Hemisphere light shifts: warm white (day) → cool blue (night)
- Building window emissive ramps up at dusk, full bright at night
- Street lamps gain PointLight at night (low intensity, warm color)

### 5.2 Sky Upgrade

**Current:** Flat `scene.background = 0x87b8e0`.

**New:** ShaderMaterial sky dome with:
- Gradient: horizon color ↔ zenith color (shifts with time of day)
- Sun disc (bright emissive circle tracking directional light)
- Star field at night (random dots on upper hemisphere)
- Cloud layer: translucent white planes at altitude, slowly drifting

### 5.3 Post-Processing

- **Bloom**: Neon signs, street lamps, car headlights glow
- **SSAO**: Ambient occlusion for depth in tight spaces between buildings
- **Tone mapping**: Already using ACES Filmic — may increase exposure at night

## Phase 8: Performance (Priority: HIGH — do alongside other phases)

### 8.1 InstancedMesh Migration
- Trees, lamps, hydrants, bollards, benches → InstancedMesh
- One draw call per prop type
- Target: <50 total draw calls for entire city

### 8.2 Geometry Merging
- Merge all road planes into single BufferGeometry
- Merge all sidewalk planes
- Merge all curb geometry

### 8.3 LOD System
- Buildings: simpler box beyond 100m
- Trees: billboard sprites beyond 80m
- Props: hide beyond 60m

### 8.4 Shadow Improvements
- Desktop: 2048×2048 shadow map (currently 1024)
- Cascaded shadow maps for quality at distance
- Mobile medium/high: enable basic shadows
